# Shared memory index

- [Atomic commits and full-solution build/test](feedback_build_and_commit.md) ‚Äî commit each logical change; always run `dotnet build` and `dotnet test` with no params
- [Never accept IConfiguration as a parameter](feedback_iconfiguration_parameter.md) ‚Äî use `AddOptions<T>().BindConfiguration("Section")`; expose only the section name (defaulted, last)
- ["SDK reference" means FrameworkReference, not the Sdk attribute](feedback_sdk_reference.md) ‚Äî add `<FrameworkReference Include="..." />`, never swap the project's `Sdk=`
- [No EF navigation properties; relationships on principal side](feedback_no_ef_navigation_properties.md) ‚Äî entities expose scalars + FK columns only; relationship config lives on the referenced entity, giving a "who points at me" view per principal
- [ISaveChangesHook order is undefined](feedback_save_changes_hook_order.md) ‚Äî hooks run in undefined order; order-dependent hooks belong combined into one hook
- [No logic in entity property setters](feedback_no_logic_in_entity_properties.md) ‚Äî entity properties stay plain; validation and normalization live in `ISaveChangesHook` implementations
- [Wait for approval after opinion questions](feedback_wait_on_opinion_questions.md) ‚Äî "what do you think?" / "perhaps we should‚Ä¶" ‚Üí give opinion and stop, even in auto mode
- [Prefer collection expressions](feedback_collection_expressions.md) ‚Äî use `[a, b]` over `ImmutableArray.Create`, `new List<>()`, `new[] { ... }`, etc.
- [Always use DateTimeOffset](feedback_always_datetimeoffset.md) ‚Äî every timestamp is `DateTimeOffset`; convert at the boundary when an API returns `DateTime`
- [Source-generated logging is the default](feedback_source_generated_logging.md) ‚Äî use `[LoggerMessage]` partial methods; direct `ILogger.LogX` is a code smell (CA1873)
- [Don't test log output](feedback_dont_test_logs.md) ‚Äî tests don't assert on log messages/levels/properties; only requirement is "doesn't throw"
- [Every bug gets a regression test](feedback_regression_test_per_bug.md) ‚Äî when a bug is found, the fix carries a test that fails on the bug and passes on the fix
- [Tests focus on public contracts only](feedback_test_public_contracts_only.md) ‚Äî exercise interfaces, not internals; no `InternalsVisibleTo`, no assertions on impl-specific details. Substitute deps via standard BCL seams (`TimeProvider`, etc.).
- [Loud failures beat silent success](feedback_loud_failures_over_silent_success.md) ‚Äî when a path can only be hit by a bug, surface the failure (403/exception/etc.) instead of falling through to a sentinel. Distinguish "didn't try" from "tried and failed".
- [Deferred ‚Äî IShearsPaths refactor + config.json pre-DI lookup](project_deferred_paths_refactor.md) ‚Äî pulled out of bearer-auth thread on 2026-04-28; pick up 2026-04-29. Static `LlamaShearsPaths` ‚Üí injectable `IShearsPaths`; decide config.json fate (drop vs inline lookup) before the sweep.
- [Vector store choice ‚Äî Microsoft.Extensions.VectorData](project_vector_store.md) ‚Äî when vector storage is added, use `Microsoft.Extensions.VectorData` with the SQLite (sqlite-vec) connector; not yet implemented
- [Self-registering DI helpers](feedback_self_registering_di_helpers.md) ‚Äî per-item registrations call their companion infrastructure registration so callers can't forget to wire the consumer
- [Provider.Abstractions is the core](project_provider_abstractions_is_core.md) ‚Äî `Provider.Abstractions` is the de-facto plugin contract; will become NuGet-shipped (AssemblyLoadContext); don't rename or split preemptively
- [Agent config lifecycle](project_agent_config_lifecycle.md) ‚Äî disk ‚Üí `AgentManager` (lifecycle, scan top-level `agents/*.json` on tick) ‚Üí agent provider (read API); configs are immutable snapshots; in-flight interactions see one snapshot end-to-end
- [MCP is the tool security boundary](project_mcp_is_tool_security_boundary.md) ‚Äî agents have shell access; tool allow/deny is enforced in MCP `WithListToolsHandler`/`WithCallToolHandler` only. Agent loop is control flow, never security. Route slug is convenience, not principal.
- [MCP principal will be a bearer-shaped in-process nonce](project_mcp_principal_nonce_plan.md) ‚Äî slug is today's placeholder principal; future is a single-use, agent-bound nonce carried as `Authorization: Bearer`, validated by an ASP.NET Core auth handler that emits a `ClaimsPrincipal`. Tools/policy read `HttpContext.User`. Don't build yet.
- [MCP client loopback auth handler](project_mcp_client_loopback_auth_handler.md) ‚Äî agent-side outbound MCP `HttpClient` uses a `DelegatingHandler` that detects loopback-to-our-listener and injects the current agent's bearer; external destinations untouched. Future, not yet built.
- [Framework-first is intentional](project_framework_first_is_intentional.md) ‚Äî substrate-before-agent-loop is deliberate sequencing to avoid retrofit debt; don't critique scaffolding-to-feature ratio
